# Product Requirements Document (PRD)

## 1. 제품 개요

**제품명**: 온라인 교육 플랫폼 수강 신청 시스템

**목적**: 크리에이터(강사)가 강의를 개설하고, 클래스메이트(수강생)가 수강 신청을 할 수 있는 온라인 교육 플랫폼입니다.

---

## 2. 사용자 스토리

### 2.1 페르소나 (Primary Persona)

**이름**: 김지민 (크리에이터)
- **역할**: 개발 강사
- **목표**: 자신의 전문 지식을 공유하고 수익을 창출
- **고민사항**:
  - 강의 내용을 어떻게 구성할지 고민
  - 적절한 가격 책정이 어려움
  - 수강 정원을 관리하기 어려움
- **행동 패턴**:
  1. 강의 계획을 세움
  2. 플랫폼에 강의 등록
  3. 수강 신청 현황 모니터링
  4. 수강생들과 소통

### 2.2 세컨더리 Persona

**이름**: 이수진 (클래스메이트)
- **역할**: 신입 개발자
- **목표**: 실무에 바로 적용할 수 있는 기술 습득
- **고민사항**:
  - 강의가 실용적인지 고민
  - 가격 부담
  - 일정과 시간 맞는지 확인
- **행동 패턴**:
  1. 플랫폼에서 강의 탐색
  2. 리뷰 및 커리큘럼 참고
  3. 수강 신청
  4. 결제 및 수강

---

## 3. 기능 명세

### 3.1 1단계: 강의 선택

**필수 기능**:
- [x] 강의 목록 조회
  - 카테고리 필터링 (development, design, marketing, business)
  - 카드 UI로 강의 목록 표시
- [x] 강의 정보 표시
  - 제목, 설명, 가격
  - 강의 기간 (시작일 ~ 종료일)
  - 강사 이름
  - 현재 신청 인원 / 정원
- [x] 신청 유형 선택
  - 개인 신청: 1인 수강
  - 단체 신청: 2~10인 그룹 수강
- [x] 상태별 필터링
  - DRAFT: 초안 (신청 불가)
  - OPEN: 모집 중 (신청 가능)
  - CLOSED: 모집 마감 (신청 불가)

**UI 요구사항**:
- 선택된 강의 하이라이트 표시
- 정원 거의 임박 (currentEnrollment/maxCapacity)
- 정원 초과 시 "마감 임박" 표시

### 3.2 2단계: 수강생 정보 입력

**공통 필드** (개인/단체 공통):
- [x] **이름** (필수, 2~20자)
- [x] **이메일** (필수, 이메일 형식 검증)
- [x] **전화번호** (필수, 한국 형식: `01X-XXXX-XXXX`)
- [x] **수강 동기** (선택, 최대 300자)

**단체 신청 추가 필드**:
- [x] **단체명** (필수)
- [x] **신청 인원수** (필수, 2~10인)
- [x] **참가자 명단** (인원수만큼 입력)
  - 각 참가자: 이름, 이메일
- [x] **담당자 연락처** (필수)

**UI 요구사항**:
- 실시간 유효성 검증 (blur 시)
- 에러 메시지 필드 바로 표시
- 단체 신청 시 인원수 변경에 따라 참가자 필드 동적 추가/제거
- 참가자 최소 2명, 최대 10명

### 3.3 3단계: 확인 및 제출

**필수 기능**:
- [x] 입력 내용 요약 표시
  - 1단계: 강의 정보
  - 2단계: 신청자 정보
  - 총 결제 금액 (단체: 가격 × 인원수)
- [x] 각 섹션별 "수정" 링크
- [x] 이용약관 동의 체크박스
- [x] 제출 버튼
- [x] 제출 결과 화면
  - 신청 번호 표시
  - 신청 정보 요약
  - 인쇄하기 버튼

**UI 요구사항**:
- 이탈 방지 (beforeunload 이벤트)
- 로딩 상태 표시
- 에러 발생 시 알림 + 재시도 가능

## 4. UI/UX 요구사항

### 4.1 스텝 인디케이터

**현재 진행 단계 시각화**:
```
●───●───○      (1단계 완료, 2단계 진행 중)
```

**상태**:
- 완료: ✓ (체크 표시)
- 진행 중: ○ (숫자 표시)
- 대기: ○ (숫자 표시)

### 4.2 유효성 에러 UX

**에러 표시**:
- 필드 바로 빨간 박스 테두리
- 에러 메시지: 붉은색 텍스트
- 자동 포커스 이동

**실시간 검증**:
- blur 시점에 해당 필드만 검증
- 다음 단계로 넘어갈 때 전체 검증

### 4.3 조건부 필드 전환

**개인 → 단체 전환**:
- group 데이터 초기화 (confirm 대화상자 없이 자동)
- 단체 필드 표시

**단체 → 개인 전환**:
- group 데이터 유지 (나중에 다시 단체로 전환 가능)

### 4.4 반응형 레이아웃

**모바일** (최대 768px):
- 세로 스크롤 레이아웃
- 카드 단일 컬럼

**태블릿/데스크톱**:
- 2-3 컬럼 그리드
- 수평형 레이아웃

## 5. 와이어프레임

### 5.1 해피 스토리

```
1. 홈페이지 → 수강 신청 페이지
   ↓
2. 강의 선택 (카테고리 필터)
   ↓
3. 개인/단체 선택
   ↓
4. 수강생 정보 입력
   ↓
5. 이용약관 동의
   ↓
6. 제출
   ↓
7. 신청 완료 화면
   ↓
8. 이메일 확인 (백엔드에서 발송)
```

### 5.2 이탈 지점

| 단계 | 이탈 지점 | 이탈 시 데이터 |
|------|-----------|--------------|
| 1단계 | 강의 선택 미완료 | sessionStorage에 자동 저장 |
| 2단계 | 필수 정보 미입력 | sessionStorage에 자동 저장 |
| 3단계 | 약관 미동의 | sessionStorage에 자동 저장 |
| 완료 화면 | - | sessionStorage 정리 |

### 5.3 예외 시나리오

**정원 초과**:
- "정원이 초과되었습니다. 다음 기회를 이용해 주세요."
- 해당 강의 카드에 "마감 임박" 뱃지 표시

**중복 신청**:
- "이미 신청한 강의입니다."
- 409 Conflict 에러

**결제 실패**:
- "결제 처리 중 오류가 발생했습니다. 다시 시도해 주세요."
- 입력 데이터 유지, 재시도 가능

## 6. 비기능 요구사항

### 6.1 성능

- 페이지 로드 시간: 2초 이내
- 폼 전환 시간: 300ms 이내
- API 응답 시간: 1초 이내 (Mock 환경)

### 6. 호환성

- **브라우저**: Chrome 90+, Firefox 88+, Safari 14+
- **해상도**: 1024px 이상 권장 (모바일 지원)
- **OS**: Windows 10+, macOS 11+, Linux

### 6. 접근성

- 키보드 네비게이션 지원
- 스크린 리더기 호환
- 적절한 색 대비 준수 (WCAG 2.1 AA)

---

## 7. 성공 지표

### 7.주요 지표 (KPI)

- **전환율**: 1단계 → 3단계 70% 이상
- **완료율**: 최종 제출 60% 이상
- **만족도**: 사용자 경험 만족도 4.5/5.0 이상

### 7.2 기술 지표

- **테스트 커버리지**: 80% 이상
- **번들 크기**: Main 번들 200KB 이하
- **Lighthouse 점수**: 90 이상
